## 設計原則

**卡皮教練的知識庫不是百科全書。**

LLM 本身已具備增肌減脂、三鐵訓練的基礎知識。  
知識庫只補充三件事：

1. **LLM 不知道的**：台灣在地賽事、台灣健身房文化的現實
2. **容易答錯的**：邊界判斷、免責聲明的觸發時機
3. **卡皮教練獨有的**：他的評估框架、他對學員的標準處理流程

**物理治療的邊界原則（核心設計決策）：**

> 卡皮教練的專業包含認識邊界。  
> 遇到受傷或身體症狀，他說的是「這個症狀要去看物理治療師」，  
> 不提供具體的復健動作，不診斷傷況，不說「你應該是 XXX 傷」。  
> 他可以說「在確認之前，這些動作可以暫時避免」，但不替代專業評估。

### `boundaries/bnd_001_injury_referral.md`（最優先建立）

這個檔案定義卡皮教練的邊界行為，Claude Code 在實作 system prompt 時必須引用。

```markdown
# 卡皮教練的受傷邊界處理指南

## 核心原則
卡皮教練的專業包含認識自己的邊界。
遇到受傷或身體症狀，他提供支持與方向，但不提供診斷或復健處方。

## 觸發條件（以下任一出現，立即進入邊界模式）
- 用戶描述疼痛（痛、酸、麻、刺、腫、不舒服）且持續超過一次訓練
- 用戶提到曾經受傷的部位再度有感覺
- 用戶描述訓練中突然出現的劇烈疼痛
- 用戶詢問「我這樣算受傷嗎」

## 邊界模式的標準回應結構

1. 先承接（不忽略情緒）
   「練到這種感覺，繼續撐下去確實很難。」

2. 說清楚邊界（不診斷）
   「這個症狀卡皮沒辦法替你判斷是什麼傷，
    需要讓物理治療師實際評估才能確定。」

3. 給一個當下可做的事（具體但保守）
   「在確認之前，建議先停掉會引發這個感覺的動作，
    不要等它痛到無法訓練才處理。」

4. 必要時給台灣資源方向（不指定診所）
   「可以找運動醫學科或物理治療師，
    台灣各大醫院都有運動醫學門診。」

## 絕對禁止
- 說「你應該是 XXX 傷」（韌帶、肌腱、骨折等任何診斷詞）
- 提供具體的復健動作序列
- 說「你可以繼續練，只是要注意 XXX」（任何讓用戶忽視症狀的建議）
- 推薦消炎藥、貼布品牌或任何藥物

## 可以說的事（邊界內）
- 「這個部位容易在 XXX 動作過量時出現不適，
   你最近這個動作的量有增加嗎？」（協助用戶自我觀察，不診斷）
- 「在物理治療師評估之前，上肢的訓練可以照常，
   下肢先暫停比較安全。」（保守的暫時替代方向，不是復健處方）
```

### `triathlon/tri_001_beginner_roadmap.md`

```markdown
# 初鐵完賽路線圖

## 適用對象
目標：完成人生第一場三鐵（建議從 51.5 奧運距離開始）
時間：距離比賽 16–20 週

## 三個訓練支柱
游泳 / 騎車 / 跑步，三項都要練，但優先順序因人而異：
- 游泳最弱的人：游泳佔週訓練量 40%，其餘各 30%
- 三項均衡的人：各 33%，長週末做 Brick

## 16 週架構（四個 4 週區塊）
Week 1–4：建立基礎（低強度，建立習慣與技術）
Week 5–8：提升量（加入 Brick，增加長距離）
Week 9–12：強化強度（加入間歇，模擬比賽配速）
Week 13–15：降量（Taper，保持感覺，不做新刺激）
Week 16：比賽週

## 最重要的一件事
初鐵的目標是完賽，不是時間。
配速保守比撞牆好，第一場留下好印象比較重要。
```

### `triathlon/tri_002_zone_training.md`

```markdown
# 心率區間訓練（Zone Training）

## 為什麼三鐵選手要做 Zone 2
三鐵是耐力運動。Zone 2 訓練建立有氧基礎，
讓你在比賽中可以長時間維持效率，不燒盡。

## 五個區間（以最大心率 %HRmax 為基準）
Zone 1：< 60%  極輕鬆，暖身或主動恢復
Zone 2：60–70%  可以說完整句子，這是訓練量的主力
Zone 3：70–80%  有些喘，可以說短句
Zone 4：80–90%  很喘，是高強度間歇的區間
Zone 5：> 90%  全力衝刺，無法說話

## 估算最大心率
公式（粗估）：220 − 年齡
更精確：做一次場地測驗或使用跑錶的自動偵測

## 初學者的常見錯誤
大部分人練 Zone 2 時其實跑太快，落在 Zone 3。
檢查方式：能不能跟旁邊的人輕鬆聊天？
可以 → Zone 2。喘到說不完整句子 → 放慢。

## 卡皮教練的建議
週訓練量的 70–80% 放在 Zone 2，
其餘 20–30% 才是高強度。
大部分人的問題不是練太少，是練太快。
```

### `triathlon/tri_003_brick_workout.md`

```markdown
# 磚塊訓練（Brick Workout）

## 什麼是 Brick
連續完成兩項訓練，中間不休息（最常見：騎車接跑步）。
名稱由來：騎完接著跑，腳感覺像磚頭一樣重。

## 為什麼要練 Brick
三鐵轉換是身體需要適應的技能。
騎完換跑，肌肉使用模式完全不同，需要練習切換。

## 標準 Brick 格式（初學版）
騎車 45–60 分鐘（Zone 2）→ 立刻換跑步鞋 → 跑 15–20 分鐘
重點不是距離，是體驗那個切換的感覺。

## 建議頻率
16 週計畫中，Week 5 開始，每週一次。
比賽前 4 週可增加到每週兩次。

## 注意事項
Brick 的疲勞累積比單項訓練更高。
做完 Brick 的隔天安排輕鬆訓練或休息。
```

### `triathlon/tri_005_nutrition_race_day.md`

```markdown
# 比賽日補給策略

## 基本原則
比賽日不是測試新食物的時候。
所有補給品都要在訓練中測試過。

## 比賽前一天
晚餐：熟悉的高碳水食物（米飯、麵食），不要嘗試新餐廳
睡前：確認補給包準備好，不要因為緊張而過量進食

## 比賽當天早餐
比賽前 2–3 小時：300–500 卡，以碳水為主
例：白吐司 + 香蕉 + 運動飲料（熟悉且好消化）

## 騎車段補給（最重要）
每 20–30 分鐘補充一次，不要等到餓
能量膠 or 能量棒 + 電解質飲料
目標：每小時攝取 60–90g 碳水

## 跑步段補給
每個補給站都要停下來補充（不要跑過去）
可以接受固體食物（香蕉切片是常見選項）
注意電解質，特別是鹽分

## 常見錯誤
- 騎車段吃太少，跑步段沒有能量
- 喝太多純水（低鈉血症風險）
- 嘗試比賽當天才第一次用的補給品
```

### `strength/str_001_progressive_overload.md`

```markdown
# 漸進超負荷原則

## 這是什麼
肌肉成長需要持續的新刺激。
如果每次訓練都做一樣的重量和次數，身體適應後就不再進步。

## 實際執行方式
方法 1：每次訓練增加重量（5–10%）
方法 2：相同重量增加次數或組數
方法 3：縮短組間休息時間
方法 4：提升動作控制品質（速度、範圍）

## 新手的漸進速度
新手期（前 3–6 個月）：每週都可以進步，不要浪費這個階段
進階後：每 2–4 週進步一次是正常的

## 卡皮教練的提醒
漸進超負荷不等於每次都要破 PR。
睡眠不足、壓力大、飲食不夠的情況下，
維持現有重量完成高品質動作，比強行加重更值得。
```

### `strength/str_002_beginner_program.md`

```markdown
# 新手肌力訓練框架（三日全身）

## 適用對象
完全新手，或超過半年沒有重訓的人

## 週課表安排
一 / 三 / 五（或任何間隔一天的三天）

## 每次訓練的基本結構
暖身 10 分鐘 → 主項目 → 輔助動作 → 緩和伸展

## 必學的六個基本動作模式
1. 深蹲模式（深蹲、腿推機）
2. 髖鉸鏈（硬舉、羅馬尼亞硬舉）
3. 水平推（臥推、伏地挺身）
4. 水平拉（坐姿划船、彈力帶划船）
5. 垂直推（肩推）
6. 垂直拉（引體向上、高拉機）

## 新手前三個月的核心原則
重量不重要，動作品質最重要。
寧可輕重量做對，不要重重量做錯。
受傷是最昂貴的訓練成本。
```

### `strength/str_003_deload_week.md`

```markdown
# 減量週（Deload Week）

## 什麼是減量週
刻意降低訓練量和強度，讓身體充分恢復。
不是偷懶，是訓練計畫的一部分。

## 何時需要
- 連續高強度訓練 4–6 週後
- 比賽前 1–2 週（Taper）
- 身體持續感到疲勞、動力下降、睡眠品質變差

## 減量週的做法
重量維持 60–70%，組數和次數減少 40–50%
繼續訓練，但不追求強度

## 常見誤解
「減量週會讓我退步。」→ 錯。超補償發生在恢復期，減量後通常表現更好。
「我狀態很好，不需要減量。」→ 感覺好的時候更要按計畫減量，等感覺很差才減量已經太晚。
```

### `fat_loss/fat_001_tdee_calculation.md`

```markdown
# TDEE 計算與熱量赤字設定

## TDEE 是什麼
Total Daily Energy Expenditure，每日總消耗熱量。
你需要吃超過這個數字才會增重，低於才會減重。

## 粗估 TDEE
基礎代謝率（BMR）× 活動係數

BMR 公式（Mifflin-St Jeor）：
男：(10 × 體重 kg) + (6.25 × 身高 cm) − (5 × 年齡) + 5
女：(10 × 體重 kg) + (6.25 × 身高 cm) − (5 × 年齡) − 161

活動係數：
久坐（幾乎不運動）：× 1.2
輕度活動（每週 1–3 次）：× 1.375
中度活動（每週 3–5 次）：× 1.55
高度活動（每週 6–7 次）：× 1.725

## 減脂的赤字設定
每週減 0.5–1% 體重是安全且可持續的速度
建議赤字：每日 300–500 卡（不要超過 500）
赤字太大 → 掉肌肉 + 代謝適應 + 難以維持

## 卡皮教練的提醒
TDEE 只是估算，每個人的代謝不同。
連續兩週沒有體重變化，才考慮調整飲食，
不要因為一天的體重波動就改變計畫。
```

### `fat_loss/fat_002_body_recomp.md`

```markdown
# 增肌減脂同步（Body Recomposition）

## 誰可以做到
1. 完全新手（前 3–6 個月效果最明顯）
2. 有相當長時間沒有訓練的人
3. 體脂率明顯偏高的人

## 怎麼做
熱量：維持 TDEE 附近（不大幅赤字，不大幅盈餘）
蛋白質：每公斤體重 1.6–2.2g（這是最重要的單一變數）
訓練：一定要有肌力訓練，不能只靠有氧

## 進展速度
比單純增肌或單純減脂都慢。
這是取捨，接受它或選擇先做其中一件事。

## 何時選擇分開做
體脂率偏低（男 < 12%，女 < 20%）→ 先增肌再減脂
急需快速看到體態變化 → 先減脂效果更明顯
```

### `recovery/rec_001_sleep_and_adaptation.md`

```markdown
# 睡眠與訓練超補償

## 為什麼睡眠比訓練本身更重要
肌肉不是在訓練時生長，是在休息時生長。
訓練是給身體壓力，睡眠是身體修復和適應的時間。

## 睡眠不足對訓練的影響
- 肌力下降 10–30%
- 恢復速度變慢
- 受傷風險上升
- 食慾調節失常（更容易過食）

## 最低需求
每晚 7–9 小時。
運動員在高訓練量期間建議 8–9 小時。

## 卡皮教練的立場
如果你只能選一件事：睡好還是多練一次訓練？
選睡好。這不是廢話，這是生理學。

## 實用建議
- 訓練時間盡量避免睡前 3 小時內（影響入睡）
- 睡前降低螢幕藍光曝露
- 保持固定的睡醒時間比睡前儀式更重要
```

### 觸發方式（直接 lookup，無向量搜尋）

```python
def search_fitness_knowledge(query: str, domain: str) -> str:
    """
    根據 domain 直接讀取對應的知識庫檔案。
    回傳相關內容，注入 system prompt。

    domain 對應：
    triathlon  → knowledge_base/triathlon/ 所有檔案
    strength   → knowledge_base/strength/ 所有檔案
    fat_loss   → knowledge_base/fat_loss/ 所有檔案
    recovery   → knowledge_base/recovery/ 所有檔案
    injury     → knowledge_base/boundaries/bnd_001_injury_referral.md（必讀）
    general_fitness → 不注入額外知識，依賴 LLM 本身

    boundaries/bnd_002_medical_disclaimer.md 在觸發詞出現時自動注入，
    不依賴 domain 判斷。
    """
```

### 注入優先順序

1. `bnd_001_injury_referral.md`（只要提到受傷，最優先）
2. Domain 對應的知識庫檔案
3. `athlete_profile.md`（永遠注入，提供個人脈絡）
4. `training_plan.md`（只在 COACHING 狀態注入）

### Token 預算

- 單次注入的知識庫內容上限：800 tokens
- 超過上限時，只注入最相關的一個檔案
- `athlete_profile.md` 不計入此上限（永遠完整注入）
